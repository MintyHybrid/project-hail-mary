6.1 Introduction

Horizontal gene transfer (HGT) across kingdoms is the most parsimonious explanation for the conservation pattern identified in Chapter 1 (The Discovery). This chapter quantifies phylogenetic incongruence and characterises the transferred locus using flanking-sequence alignments from eukaryotic genomes.

6.3.4 Strawberry (Fragaria) — presence/absence of the ~3 kb fragment

NoteWhich virus, and is strawberry even a host?

The strawberry loci here were recovered by BLASTing the DNAzyme-flanking sequence of one specific isolate — NC_003377.1, Maize dwarf mosaic virus (MDMV) — against plant genomes. MDMV is a maize/grass-infecting potyvirus, and strawberry is not a documented host of MDMV. So the Fragaria match is a cross-kingdom sequence homology to the viral flank, not evidence that MDMV infects strawberry. Read the section below as “a genomic fragment in strawberry happens to match the MDMV DNAzyme-flank region”, with the origin of that match (ancient HGT vs coincidence vs a shared host gene) still open.

The viral-homologous insertion in strawberry is a long (~3 kb) fragment, not just the 18-mer. strawberryseqs.fasta holds only fragment-positive Fragaria loci, so on their own they cannot separate a single ancient insertion from repeated transfer or lineage-specific loss. To test this, the homologous ~3 kb locus is retrieved from related Fragaria genomes by BLASTing a representative positive locus against Fragaria on NCBI (scripts/fetch_strawberry_homologs.R, cached to data/strawberry_homologs.fasta + data/strawberry_homolog_table.tsv). For each related genome we record query coverage over the 3 kb locus: near-complete coverage → the fragment is present; coverage only over the flanks → absent (a deletion where the insert sits). Presence/absence is then mapped onto the host phylogeny.

What the broadened BLAST reveals. Searching the ~3 kb positive locus against the whole Rosaceae (scripts/strawberry_broad_blast.R, cached to data/strawberry_broad_blast_hsps.tsv; corroborated by a 250-HSP megablast of 82 Fragaria/Rosaceae subjects in data/strawberry_megablast_hittable.txt) shows the homology is confined to the two ends of the query, with a central gap — exactly the pattern expected if a lineage-specific insert sits between two host-derived flanks:

  • 5′ flank (~bp 1–670): conserved genomic sequence, recovered from Malus domestica and Fragaria chromosomes (~82 % identity).
  • central ~1.4 kb (~bp 670–2130): no Rosaceae counterpart — the candidate horizontally-acquired / lineage-specific segment.
  • 3′ flank (~bp 2130–3018): the host TCP7 transcription-factor gene, matching TCP7 (and its paralogues) across Fragaria, Rosa, Prunus, Malus, Pyrus and Potentilla.

The many partial (~28–35 % coverage) hits spread across numerous Fragaria chromosomes are TCP-family paralogues matching only the 3′ TCP7 flank, not the insert. The most parsimonious reading is therefore that the ~3 kb element is a host locus (a genomic region fused to TCP7) carrying an insert present only in a subset of Fragaria lineages and absent from otherwise closely related genomes — consistent with a lineage-specific acquisition rather than a universally conserved host feature.

6.3.5 Localizing the insert within the 3 kb (sliding window)

Horizontal acquisition, if any, is expected to sit in a sub-region of the 3 kb, not throughout. Using the combined alignment, we scan along a representative signature-positive locus and, in sliding windows, measure two tracks: conservation among the positive loci and identity to the TCP7 relatives. The host TCP7 region should score high on both; a lineage-specific insert should be conserved among positives but have no counterpart in the relatives (the track drops to zero).

Code 
mm      <- as.matrix(as(straw_aln, "DNAStringSet"))   # aligned char matrix
rn      <- rownames(mm)
pos_rows <- grep("^pos_", rn); hom_rows <- grep("^hom_", rn)
ref_row <- pos_rows[1]
refv    <- mm[ref_row, ]
ref_cols <- which(refv != "-")                        # coordinates along the ref locus

frac_match <- function(rows) {
  if (length(rows) == 0) return(rep(NA_real_, ncol(mm)))
  ref_mat <- matrix(refv, nrow = length(rows), ncol = length(refv), byrow = TRUE)
  colMeans(mm[rows, , drop = FALSE] == ref_mat)
}
wavg <- function(x, w = 90) as.numeric(stats::filter(x, rep(1 / w, w), sides = 2))

track <- tibble::tibble(
  pos = seq_along(ref_cols),
  `Conservation among positive loci`  = wavg(frac_match(setdiff(pos_rows, ref_row))[ref_cols]),
  `Identity to TCP7 relatives`        = wavg(frac_match(hom_rows)[ref_cols])) %>%
  tidyr::pivot_longer(-pos, names_to = "track", values_to = "value")

TipProposed next steps (not yet run)

Working hypothesis: if there was any cross-kingdom transfer, it is most parsimoniously a single event propagated through strawberry breeding (the fragment appears in several Fragaria accessions). To test this properly:

  1. Delimit the insert precisely. Pin down the exact boundaries of the central non-TCP7 segment (the candidate insert), rather than the ~670–2130 bp estimate above.
  2. Re-BLAST the whole insert against viruses. The initial search used only the 18-mer. If the fragment is genuinely MDMV-derived, the full insert (not just the 18-mer) should also match the MDMV genome; if it does not, a different (or no) viral origin is more likely.
  3. Fragment-internal phylogeny vs breeding history. Build a tree from the large strawberry fragments alone and ask whether its topology tracks documented Fragaria breeding/domestication — a single-introduction-then- inheritance model predicts congruence.

These need live NCBI BLAST and curated breeding metadata, so they are recorded here as next steps rather than executed.

6.4 Coordinate utilities

Helper functions for converting NCBI-style FASTA headers to GRanges.

Code 
library(GenomicRanges)
library(IRanges)

extract_ncbi_coords <- function(x) {
  nms <- if (methods::is(x, "XStringSet")) names(x) else x
  pattern <- "([A-Z0-9_]+\\.[0-9]+)\\.([+-]):(-?[0-9]+)-(-?[0-9]+)"
  m <- regexec(pattern, nms)
  matches <- regmatches(nms, m)
  do.call(rbind, lapply(matches, function(x) {
    if (length(x) == 5) {
      data.frame(full_name = x[1], seqname = x[2], strand = x[3],
                 start = as.integer(x[4]), end = as.integer(x[5]),
                 stringsAsFactors = FALSE)
    } else {
      data.frame(full_name = paste0(x), seqname = NA, strand = NA,
                 start = NA, end = NA, stringsAsFactors = FALSE)
    }
  }))
}

biostring_to_granges <- function(dna) {
  if (!methods::is(dna, "XStringSet"))
    stop("Input must be a DNAStringSet or other XStringSet")
  df <- extract_ncbi_coords(dna)
  gr <- GRanges(
    seqnames = df$seqname,
    ranges   = IRanges(start = df$start, end = df$end),
    strand   = df$strand)
  mcols(gr)$sequence <- dna
  gr
}

### Strawberry (*Fragaria*) — presence/absence of the ~3 kb fragment

::: {.callout-note}
## Which virus, and is strawberry even a host?
The strawberry loci here were recovered by BLASTing the **DNAzyme-flanking
sequence of one specific isolate — NC_003377.1, *Maize dwarf mosaic virus*
(MDMV)** — against plant genomes. MDMV is a **maize/grass-infecting potyvirus**,
and **strawberry is *not* a documented host of MDMV**. So the *Fragaria* match is
a **cross-kingdom sequence homology to the viral flank**, *not* evidence that
MDMV infects strawberry. Read the section below as "a genomic fragment in
strawberry happens to match the MDMV DNAzyme-flank region", with the origin of
that match (ancient HGT vs coincidence vs a shared host gene) still open.
:::

The viral-homologous insertion in strawberry is a **long (~3 kb) fragment**, not
just the 18-mer. `strawberryseqs.fasta` holds only fragment-**positive**
*Fragaria* loci, so on their own they cannot separate a single ancient insertion
from repeated transfer or lineage-specific loss. To test this, the **homologous
~3 kb locus is retrieved from related *Fragaria* genomes** by BLASTing a
representative positive locus against *Fragaria* on NCBI
(`scripts/fetch_strawberry_homologs.R`, cached to
`data/strawberry_homologs.fasta` + `data/strawberry_homolog_table.tsv`). For each
related genome we record **query coverage** over the 3 kb locus: near-complete
coverage → the fragment is *present*; coverage only over the flanks → *absent*
(a deletion where the insert sits). Presence/absence is then mapped onto the host
phylogeny.

**What the broadened BLAST reveals.** Searching the ~3 kb positive locus against
the whole **Rosaceae** (`scripts/strawberry_broad_blast.R`, cached to
`data/strawberry_broad_blast_hsps.tsv`; corroborated by a 250-HSP megablast of
82 *Fragaria*/Rosaceae subjects in
`data/strawberry_megablast_hittable.txt`) shows the homology is **confined to the
two ends of the query, with a central gap** — exactly the pattern expected if a
lineage-specific insert sits between two host-derived flanks:

- **5′ flank (~bp 1–670):** conserved genomic sequence, recovered from *Malus
  domestica* and *Fragaria* chromosomes (~82 % identity).
- **central ~1.4 kb (~bp 670–2130):** **no Rosaceae counterpart** — the candidate
  horizontally-acquired / lineage-specific segment.
- **3′ flank (~bp 2130–3018):** the host **TCP7 transcription-factor gene**,
  matching TCP7 (and its paralogues) across *Fragaria*, *Rosa*, *Prunus*,
  *Malus*, *Pyrus* and *Potentilla*.

The many partial (~28–35 % coverage) hits spread across numerous *Fragaria*
chromosomes are TCP-family paralogues matching only the 3′ TCP7 flank, not the
insert. The most parsimonious reading is therefore that the ~3 kb element is a
host locus (a genomic region fused to *TCP7*) carrying an insert **present only
in a subset of *Fragaria* lineages and absent from otherwise closely related
genomes** — consistent with a lineage-specific acquisition rather than a
universally conserved host feature.

### Localizing the insert within the 3 kb (sliding window)

Horizontal acquisition, if any, is expected to sit in a **sub-region** of the
3 kb, not throughout. Using the combined alignment, we scan along a
representative signature-positive locus and, in sliding windows, measure two
tracks: **conservation among the positive loci** and **identity to the TCP7
relatives**. The **host TCP7 region** should score high on *both*; a
**lineage-specific insert** should be conserved among positives but have *no*
counterpart in the relatives (the track drops to zero).

```{r strawberry-sliding}
#| fig-width: 10

ref_cols <- which(refv != "-")                        # coordinates along the ref locus

frac_match <- function(rows) {
  if (length(rows) == 0) return(rep(NA_real_, ncol(mm)))
  ref_mat <- matrix(refv, nrow = length(rows), ncol = length(refv), byrow = TRUE)

wavg <- function(x, w = 90) as.numeric(stats::filter(x, rep(1 / w, w), sides = 2))
```

## Proposed next steps (not yet run)
Working hypothesis: *if* there was any cross-kingdom transfer, it is most
parsimoniously a **single event propagated through strawberry breeding** (the
fragment appears in several *Fragaria* accessions). To test this properly:

1. **Delimit the insert precisely.** Pin down the exact boundaries of the central
   non-*TCP7* segment (the candidate insert), rather than the ~670–2130 bp
   estimate above.
2. **Re-BLAST the *whole* insert against viruses.** The initial search used only
   the 18-mer. If the fragment is genuinely MDMV-derived, the **full insert (not
   just the 18-mer) should also match the MDMV genome**; if it does not, a
   different (or no) viral origin is more likely.
3. **Fragment-internal phylogeny vs breeding history.** Build a tree from the
   large strawberry fragments alone and ask whether its topology tracks
   documented *Fragaria* breeding/domestication — a single-introduction-then-
   inheritance model predicts congruence.

## Coordinate utilities

Helper functions for converting NCBI-style FASTA headers to GRanges.

```{r coord-utils}
library(GenomicRanges)
library(IRanges)

extract_ncbi_coords <- function(x) {
  nms <- if (methods::is(x, "XStringSet")) names(x) else x
  pattern <- "([A-Z0-9_]+\\.[0-9]+)\\.([+-]):(-?[0-9]+)-(-?[0-9]+)"
  m <- regexec(pattern, nms)
  matches <- regmatches(nms, m)
  do.call(rbind, lapply(matches, function(x) {
    if (length(x) == 5) {
      data.frame(full_name = x[1], seqname = x[2], strand = x[3],
                 start = as.integer(x[4]), end = as.integer(x[5]),
                 stringsAsFactors = FALSE)
    } else {
      data.frame(full_name = paste0(x), seqname = NA, strand = NA,
                 start = NA, end = NA, stringsAsFactors = FALSE)
    }
  }))
}

biostring_to_granges <- function(dna) {
  if (!methods::is(dna, "XStringSet"))
    stop("Input must be a DNAStringSet or other XStringSet")
  df <- extract_ncbi_coords(dna)
  gr <- GRanges(
    seqnames = df$seqname,
    ranges   = IRanges(start = df$start, end = df$end),
    strand   = df$strand)
  mcols(gr)$sequence <- dna
  gr
}
```
